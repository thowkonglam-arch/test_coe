P1 — Standardised Operating Model

One Model. Every Vendor

Eliminate vendor ping-pong. One defect lifecycle. One quality language — across all 9 initiatives, simultaneously.

Unified Defect Lifecycle

Raised → Triaged → Fixed → Closed. Any team. Any tool. Same process — no exceptions.

Common Test Taxonomy

UT, SIT, UAT, regression, performance — one shared quality language. No translation needed between vendors.

Governance Cadence

Daily triage. Weekly QA review. Monthly tech council. Visibility at every level, every week.

Sign-off Discipline

Standard entry, exit and sign-off criteria. Consistent across every workstream, every release.

P3 Detail
P4 — Resiliency Testing

Resilience Is a Release Gate.

What happens to 500+ FPG stores if GCP goes down during CNY peak? If the answer isn't "we've tested that" — it's a risk. DR, BCP, ransomware: evidenced before go-live, not documented and hoped for.

Disaster Recovery Testing

Validate failover and recovery objectives for every critical system before go-live.

BCP Rehearsal

Store-by-store AND centralised recovery scenarios rehearsed — not just documented. What does a Store Manager do when iPOS is down?

Ransomware Simulation

Exercise detection, isolation, restore and incident response under timed conditions. ITMO C-13 compliance gate before release approval.

Offline & Fallback Modes

Validate fallback logic, circuit breakers, retry patterns and alternative business operations (Smart Cart DR path for iPOS).

P5 — Unified Tooling & Reporting

Unified tooling. One Dashboard. Full Traceability.

Nine workstreams using different tools = nine quality pictures that don't talk to each other. One standard closes that gap.

Core Tooling Stack*

ADO / JIRA + Xray TOSCA (SAP automation) Playwright (UI) JMeter / Gatling Postman / REST Assured Central Dashboard

*Tools listed are illustrative. Final selection aligned to FPG IT standards during design phase.

Tool Governance

Mandatory onboarding, shared field standards, central dashboard ownership, defect severity lock-down and evidence archiving in a common repository for every vendor on the programme.

RAG Programme Dashboard

RAG status, defect ageing, execution progress and blocked items rolled up for a single programme visibility view.

Evidence Repository

Store pass/fail evidence, screenshots and sign-off records in a central repository — audit-ready at any point in the programme.

P6 — AI & Regulatory Compliance Testing

AI & Regulatory Compliance Testing

The hardest part of scaling a test programme isn't writing test cases — it's doing it fast enough to keep up with the delivery pace. AI & automation mitigates that. Compliance evidence is built in, not bolted on.

AI-Driven Automation

Use AI & automation to generate test cases from requirements and user stories - proven in other clients and adoptable to FPG's retail context.

Regulatory Compliance Built In - PDPA,MAS TRM, PCI DSS, OWASP Top 10

Compliance testing woven into the quality engine — not retrospective validation. Every initiative needs to design & generate relevant audit-ready sign-off evidence. No last-minute audit scramble.

CI/CD Quality Gates

Framework selection, CI/CD integration, common automation standards and seed scripts — the starting point for scaled, fast-feedback automation.

Scalable Coverage

Automation coverage expands across the portfolio as each initiative matures — reducing regression cycle time and human dependency at each release.
